Q1기본 Java 메서드 시그니처(Signature)에 포함되는 것은?
  • 반환 타입
  • 메서드 이름 + 매개변수 타입/순서
  • 접근 제어자
  • throws 절
정답 보기
② 메서드 이름 + 매개변수 타입/순서
반환 타입은 시그니처에 포함되지 않는다. 반환 타입만 다른 메서드 두 개는 컴파일 에러.
Q2기본 아래 코드에서 컴파일 에러가 발생하는 이유는?
class Car {
    int speed = 100;
    static int getSpeed() { return speed; }
}
  • speed가 초기화되지 않았다
  • static 메서드는 인스턴스 변수에 접근할 수 없다
  • 반환 타입이 잘못되었다
  • getSpeed에 매개변수가 없다
정답 보기
② static 메서드는 인스턴스 변수에 접근할 수 없다
static 메서드는 클래스에 속하고, 인스턴스 변수는 객체에 속한다. 객체 없이 호출되는 static 메서드에서 인스턴스 변수를 쓰면 "non-static variable speed cannot be referenced from a static context" 에러.
Q3기본 아래 코드의 출력 결과는?
String a = new String("hello");
String b = new String("hello");
System.out.println(a == b);
System.out.println(a.equals(b));
  • true / true
  • false / false
  • false / true
  • true / false
정답 보기
③ false / true
==은 메모리 주소 비교 → new로 생성했으므로 다른 주소 → false.
.equals()는 내용 비교 → 둘 다 "hello" → true.
Q4기본 오버로딩(Overloading)의 메서드 결정 시점은?
  • 런타임 (Runtime)
  • 컴파일 타임 (Compile-time)
  • 링크 타임 (Link-time)
  • 클래스 로드 시

② 컴파일 타임
오버로딩은 컴파일러가 인자 타입을 보고 어느 메서드를 호출할지 결정한다. 오버라이딩은 런타임에 실제 객체 타입으로 결정.
Q5기본 <? super Integer>의 의미로 올바른 것은?

Q6기본 ArrayList와 LinkedList에 대한 설명으로 올바른 것은?
  • ArrayList는 중간 삽입/삭제가 빠르다
  • LinkedList는 인덱스(get) 접근이 빠르다
  • ArrayList는 배열 기반, LinkedList는 노드 기반이다
  • LinkedList는 메모리를 더 적게 사용한다
정답 보기
③ ArrayList는 배열 기반, LinkedList는 노드 기반
ArrayList: 인덱스 접근 O(1), 중간 삽입 O(n).
LinkedList: 중간 삽입 O(1), 인덱스 접근 O(n). 각 노드가 prev/next 참조를 추가로 저장하므로 메모리는 더 씀.
Q7기본 다음 중 Unchecked Exception(비검사 예외)은?
  • IOException
  • FileNotFoundException
  • NullPointerException
  • SQLException
정답 보기
③ NullPointerException
Unchecked: RuntimeException 하위 — 컴파일러가 처리 강제 안 함. (NPE, ArrayIndexOutOfBoundsException, ClassCastException 등)
Checked: Exception 하위지만 RuntimeException 아님 — try-catch 또는 throws 필수.

Q10기본 "클래스는 하나의 변경 이유만 가져야 한다"는 SOLID 원칙은?

Woof
Meow

Section 3 — 개념 서술

정의·이유·차이점을 서술하는 문제. 키워드 포함 여부가 채점 기준.

Q19중간 OOP의 3대 개념(Encapsulation, Inheritance, Polymorphism)을 각각 한 문장으로 정의하시오.
정답 보기
Encapsulation (캡슐화): 속성(데이터)을 외부에서 직접 접근하지 못하게 숨기고, 메서드를 통해서만 접근하도록 제한하는 것.

Inheritance (상속): 자식 클래스가 부모 클래스의 속성과 메서드를 물려받아 재사용하고, 추가/변경할 수 있는 것.

Polymorphism (다형성): 같은 이름의 메서드가 상황(서브클래스, 매개변수 등)에 따라 다르게 동작하는 것.
Q20중간 SOLID 5원칙 중 2개를 선택하여, 각각 원칙의 이름·정의·위반 예시를 서술하시오.
정답 보기
SRP: 클래스는 변경 이유가 하나여야 한다.
위반 예: UserSettingService가 이메일 변경과 보안 검사를 모두 담당 → 보안 정책이 바뀌어도, 이메일 정책이 바뀌어도 같은 클래스를 수정해야 함.

OCP: 확장에 열려 있고, 수정에 닫혀 있어야 한다.
위반 예: 새 도형 추가 시 AreaCalculator의 if-else를 직접 수정해야 하는 구조.

LSP: 서브클래스는 슈퍼클래스를 대체할 수 있어야 한다.
위반 예: Penguin이 Bird를 상속받아 setAltitude()를 빈 메서드로 구현 → 날 수 없는데 날 수 있는 것처럼 계약함.

ISP: 사용하지 않는 인터페이스 메서드 구현을 강제하면 안 된다.
위반 예: ArticleService에 list/write/delete가 묶여 있어서 읽기 전용 뷰도 write/delete를 구현해야 함.

DIP: 구현체가 아닌 추상(인터페이스)에 의존해야 한다.
위반 예: Foo 클래스가 Logger 구체 클래스를 직접 생성 → FileLogger로 바꾸려면 Foo 코드 수정 필요.
Q21중간 Checked Exception과 Unchecked Exception의 차이를 설명하고, 각각 예시를 하나씩 드시오.
정답 보기
Checked Exception: 컴파일러가 처리를 강제하는 예외. try-catch 또는 throws 선언이 없으면 컴파일 에러.
예: IOException, FileNotFoundException, SQLException

Unchecked Exception: RuntimeException 하위. 컴파일러가 처리를 강제하지 않음. 주로 프로그래밍 오류(null 참조, 배열 범위 초과 등).
예: NullPointerException, ArrayIndexOutOfBoundsException, ClassCastException
Q22중간 C++에서 헤더 파일(.h)이 필요한 이유를 Java와 비교하여 설명하시오.
정답 보기
C++는 컴파일러가 파일을 위에서 아래로 한 줄씩 읽으면서 번역하기 때문에, 함수를 호출하려면 호출 시점 이전에 해당 함수가 정의되거나 선언되어 있어야 한다.

헤더 파일은 함수의 시그니처(선언)만 담아두어, 실제 구현보다 먼저 컴파일러에게 "이 함수가 존재한다"는 정보를 제공한다. #include는 해당 위치에 헤더 내용을 붙여넣는 것과 같다.

Java는 컴파일 전에 프로젝트 전체를 분석하므로 함수 순서와 무관하게 동작한다.
Q23중간 Java 제네릭(Generics)의 장점 2가지를 서술하시오.
정답 보기
① 타입 안전성 (Type Safety): 컴파일 타임에 타입 오류를 검출한다. 제네릭 없이 Object를 쓰면 런타임에 ClassCastException이 발생할 수 있지만, 제네릭은 잘못된 타입을 컴파일 단계에서 막는다.

② 캐스팅 불필요: 꺼낼 때 자동으로 올바른 타입이 적용되어 명시적 캐스팅이 필요 없다. 코드가 간결해지고 캐스팅 실수로 인한 오류가 없어진다.
Q24중간 DRY 원칙이란 무엇인지 설명하고, 아래 코드가 DRY를 위반하는 이유와 개선 방법을 서술하시오.
static void printInt(Integer i)  { System.out.println("Type: Integer, Value: " + i); }
static void printStr(String s)   { System.out.println("Type: String, Value: "  + s); }
static void printDbl(Double d)   { System.out.println("Type: Double, Value: "   + d); }
정답 보기
DRY (Don't Repeat Yourself): 코드를 반복하지 말 것. 같은 로직을 여러 곳에 두면 수정 시 누락으로 버그가 생긴다.

위반 이유: 출력 형식이 동일한데 타입만 다른 메서드를 3개 작성. 형식을 바꾸려면 3곳을 모두 수정해야 함.

개선: 
static void print(Object obj) {
    System.out.printf("Type: %s, Value: %s\n",
        obj.getClass().getSimpleName(), obj);
}
제네릭 또는 Object를 이용해 하나의 메서드로 통합.

Section 4 — 코드 작성 / 수정

에러 찾기, 빈칸 채우기, 코드 직접 작성.

Q25응용 아래 조건을 만족하는 Java 클래스를 작성하시오.
조건:
· Shape 추상 클래스: getArea() 추상 메서드 포함
· Circle: Shape 상속, 반지름(radius) 필드, getArea() 구현 (pi = 3.14)
· Rectangle: Shape 상속, 너비(width)·높이(height) 필드, getArea() 구현
· main에서 Shape 배열에 Circle(5), Rectangle(3,4)를 담아 각 넓이 출력
정답 보기 
abstract class Shape {
    abstract double getArea();
}

class Circle extends Shape {
    double radius;
    Circle(double r) { radius = r; }
    @Override
    public double getArea() { return 3.14 * radius * radius; }
}

class Rectangle extends Shape {
    double width, height;
    Rectangle(double w, double h) { width = w; height = h; }
    @Override
    public double getArea() { return width * height; }
}

// main
Shape[] shapes = { new Circle(5), new Rectangle(3, 4) };
for (Shape s : shapes) {
    System.out.println(s.getArea());
}
// 출력: 78.5 / 12.0
Q26응용 아래 코드에서 컴파일 에러가 발생하는 줄을 모두 찾고 이유를 설명하시오.
class Counter {
    int count = 0;
    static int total = 0;

    static void increment() {
        count++;       // (A)
        total++;       // (B)
        System.out.println(count + " " + total);  // (C)
    }
}
정답 보기
에러 발생: (A), (C)의 count 참조

count는 인스턴스 변수. static 메서드 increment()는 클래스에 속하므로 특정 객체 없이 호출 가능. 어떤 객체의 count를 증가시켜야 할지 알 수 없으므로 컴파일 에러.

total(B)은 static 변수이므로 static 메서드에서 접근 가능 — 에러 아님.
Q27응용 아래 코드에서 컴파일 에러가 발생하는 이유를 설명하시오.
class Test {
    static void print(Object o)  { System.out.println("Object"); }
    static void print(String s)  { System.out.println("String"); }
    static void print(Integer i) { System.out.println("Integer"); }

    public static void main(String[] args) {
        print(null);   // (A)
    }
}

에러: ambiguous method call

null은 String, Integer 어느 타입으로도 볼 수 있다. 컴파일러는 String과 Integer 중 어느 오버로딩을 선택해야 할지 결정할 수 없어서 "ambiguous" 에러 발생.

해결: print((String) null) 또는 print((Integer) null)처럼 명시적 캐스팅으로 모호성 제거.
Q28응용 C++에서 이중 포인터 방식으로 N×N 2D 배열을 동적 할당하고, 값을 채운 뒤 올바르게 해제하는 코드를 작성하시오. (N=3, arr[i][j] = i*N+j)
정답 보기 
int N = 3;

// 할당
int** arr = new int*[N];
for (int i = 0; i < N; i++)
    arr[i] = new int[N];

// 값 채우기
for (int i = 0; i < N; i++)
    for (int j = 0; j < N; j++)
        arr[i][j] = i * N + j;

// 해제 (역순: 행 먼저, 그 다음 포인터 배열)
for (int i = 0; i < N; i++)
    delete[] arr[i];
delete[] arr;
해제 순서가 반드시 역순이어야 한다. arr을 먼저 delete[]하면 각 행의 주소를 잃어버려 메모리 누수 발생.
Q29응용 아래 C++ 코드의 빈칸을 채워 다형성이 동작하도록 하시오. main의 출력이 "Cow eats grass" / "Dog eats meat"가 되어야 한다.
class Animal {
public:
    _____ void eat() { cout << "Animal eats" << endl; }
};

class Cow : _____ Animal {
public:
    void eat() _____ { cout << "Cow eats grass" << endl; }
};

class Dog : public Animal {
public:
    void eat() override { cout << "Dog eats meat" << endl; }
};

int main() {
    Animal* a = new Cow();
    Animal* b = new Dog();
    a->eat();
    b->eat();
}
정답 보기
빈칸 순서대로: virtual / public / override

virtual: 부모에 붙이면 동적 바인딩 활성화. 없으면 포인터 타입(Animal)의 eat()이 호출됨.
public: C++ 상속 기본값은 private → 외부에서 접근하려면 public 명시 필요.
override: 부모의 virtual 함수를 재정의함을 명시. 오타 등 실수를 컴파일러가 잡아줌.
Q30응용 아래 Student 클래스가 Collections.sort()로 정렬되도록 Comparable을 구현하시오. 정렬 기준: 점수(score) 내림차순.
class Student {
    String name;
    int score;
    Student(String name, int score) {
        this.name = name; this.score = score;
    }
    // TODO: Comparable 구현
}
정답 보기 
class Student implements Comparable<Student> {
    String name;
    int score;
    Student(String name, int score) {
        this.name = name; this.score = score;
    }

    @Override
    public int compareTo(Student other) {
        return other.score - this.score;  // 내림차순
        // 오름차순이면: this.score - other.score
    }
}
compareTo 반환 규칙: 음수 → this가 앞, 0 → 동일, 양수 → other가 앞.
내림차순: other - this (큰 값이 앞으로 오도록).
