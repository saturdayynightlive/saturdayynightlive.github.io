Q21재출제 제네릭(Generics)을 사용하지 않았을 때의 문제점과 제네릭이 이를 해결하는 방법을 설명하시오.
정답 보기
제네릭 없는 코드의 문제: 
ArrayList list = new ArrayList();   // raw type
list.add("hello");
list.add(123);                       // 다른 타입도 들어감
String s = (String) list.get(1);    // 런타임 ClassCastException!
모든 원소를 Object로 저장 → 꺼낼 때 캐스팅 필요 → 잘못된 타입이면 런타임 에러.

제네릭으로 해결: 
ArrayList<String> list = new ArrayList<>();
list.add("hello");
list.add(123);     // 컴파일 에러 — 사전에 차단
String s = list.get(0);  // 캐스팅 불필요
타입 안전성 보장 + 캐스팅 불필요.
Q22중간 C++ 소멸자(Destructor)의 역할을 설명하고, 소멸자가 반드시 필요한 상황의 예를 드시오.
정답 보기
소멸자: 객체가 소멸될 때 자동으로 호출되는 특수 메서드. 이름은 ~ClassName().

Java의 GC와 달리 C++는 수동 메모리 관리가 필요하므로, 생성자에서 동적 할당한 메모리를 소멸자에서 해제해야 한다.

반드시 필요한 상황: 생성자에서 new로 힙 메모리를 할당한 경우. 
class MyArray {
    int* data;
public:
    MyArray(int n) { data = new int[n]; }
    ~MyArray() { delete[] data; }  // 소멸자 없으면 메모리 누수
};

Q27응용 아래 코드에서 에러를 찾고 이유를 설명하시오.

Q28응용 아래 조건을 만족하는 Java 코드를 작성하시오.
조건: Flyable 인터페이스(fly() 메서드), Swimmable 인터페이스(swim() 메서드). Duck 클래스가 둘 다 구현. main에서 Duck 객체로 fly(), swim() 호출.
정답 보기 
interface Flyable  { void fly(); }
interface Swimmable { void swim(); }

class Duck implements Flyable, Swimmable {
    public void fly()  { System.out.println("Duck flies"); }
    public void swim() { System.out.println("Duck swims"); }
}

// main
Duck d = new Duck();
d.fly();

Q29응용 아래 C++ 코드의 에러를 찾고, virtual 소멸자가 필요한 이유를 설명하시오.
class Base {
public:
    ~Base() { cout << "Base 소멸"; }
};
class Derived : public Base {
public:
    ~Derived() { cout << "Derived 소멸"; }
};

Base* p = new Derived();
delete p;  // 출력: ?
정답 보기
문제: 소멸자가 virtual이 아니므로 "Base 소멸"만 출력됨. Derived 소멸자가 호출되지 않아 Derived에서 할당한 리소스가 해제 안 됨 → 메모리 누수.

수정: Base 소멸자에 virtual 추가 
virtual ~Base() { cout << "Base 소멸"; }
virtual 소멸자 → delete 시 실제 객체 타입(Derived)의 소멸자가 먼저 호출되고, 그 다음 Base 소멸자 호출 → "Derived 소멸" + "Base 소멸".

class Product implements Comparable<Product> {
    String name;
    int price;
    Product(String name, int price) {
        this.name = name; this.price = price;
    }

    @Override
    public int compareTo(Product other) {
        if (this.price != other.price)
            return this.price - other.price;  // 가격 오름차순
        return this.name.compareTo(other.name); // 가격 같으면 이름순
    }
}
compareTo가 0을 반환하면 "같다"고 판단. 1차 기준이 같을 때 2차 기준으로 넘어가는 패턴.
