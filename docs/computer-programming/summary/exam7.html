10문제 · "○○의 정의로 옳은 것은?" — 정의의 정확성으로 구분

Q1정의 final 키워드의 의미로 옳은 것은?
  • final class는 상속이 불가능하고, final method는 오버라이딩이 불가능하며, final variable은 재할당이 불가능하다.
  • final class는 인스턴스화가 불가능하고, final method는 외부 호출이 불가능하며, final variable은 읽기 전용으로 제한된다.
  • final class는 메모리에서 자동 회수되는 클래스를 의미하고, final method는 가비지 컬렉터에 의해 소멸 직전에 실행되는 메서드이다.
  • final class는 반드시 상속을 통해서만 사용할 수 있고, final method는 서브클래스에서 반드시 오버라이딩을 해야 하는 추상 메서드이다.
정답 보기
①
final class = 상속 불가, final method = 오버라이딩 불가(상속은 가능), final variable = 재할당 불가(상수)다(Lec6).
④는 abstract 키워드의 설명으로, final과 정반대다.
Q2정의 super 키워드의 정의·규칙으로 옳은 것은?
  • 현재 객체(인스턴스) 자신을 가리키는 참조 변수로, 인스턴스 메서드나 생성자 내에서 현재 객체의 멤버에 접근할 때 주로 사용된다.
  • 부모 클래스의 멤버를 참조하는 키워드로, super()를 통해 부모 생성자를 호출할 때는 반드시 생성자의 첫 번째 줄에 위치해야 한다.
  • 상위 클래스의 계층 구조를 따라 조부모 클래스의 멤버를 직접 참조하는 키워드로, super.super.method()와 같은 형태로 중첩해서 사용할 수 있다.
  • 특정 인스턴스에 종속되지 않고 클래스 자체에 속하는 정적(static) 멤버에 직접 접근하기 위해 사용되는 전역적인 예약어(키워드)이다.
정답 보기
②
super는 부모 클래스 멤버를 참조한다. super()는 부모 생성자 호출로 반드시 생성자 첫 줄에 와야 하고, super.method()는 부모 메서드를 호출한다(Lec6).
①은 this, ③ super.super는 불가(조부모 직접 접근 금지), ④는 static의 설명이다.
Q3정의 instanceof 연산자의 정의로 옳은 것은?
  • 힙 메모리에 할당된 두 객체의 실제 내용이 동일한지(동등성)를 내부적으로 비교하여 그 결과를 boolean 타입으로 반환하는 연산자이다.
  • 특정 객체가 지정한 클래스(또는 그 슈퍼클래스/인터페이스)의 인스턴스인지 실행 시간에 검사하여 boolean 타입으로 반환하는 연산자이다.
  • 부모 클래스 타입의 참조 변수를 자식 클래스 타입으로 강제 변환(다운캐스팅)할 때 발생할 수 있는 캐스팅 오류를 방지해 주는 연산자이다.
  • 스택 메모리에 저장된 두 객체의 참조(메모리 주소)가 완전히 일치하는지(동일성)를 엄격하게 비교하여 boolean 타입으로 반환하는 연산자이다.
정답 보기
②
obj instanceof ClassName은 객체가 그 클래스(또는 슈퍼클래스)의 인스턴스인지 boolean으로 반환한다(Lec6). 예: Child c = new Child();일 때 c instanceof Parent는 true.
① .equals(), ③ 다운캐스팅, ④ ==의 설명이다.
Q4정의 업캐스팅(Upcasting)과 다운캐스팅(Downcasting)의 정의로 옳은 것은?
  • 업캐스팅은 자식에서 부모로의 자동 변환(항상 안전함)을 의미하고, 다운캐스팅은 부모에서 자식으로의 명시적 변환(ClassCastException 발생 가능)을 의미한다.
  • 업캐스팅은 부모 타입에서 자식 타입으로의 변환을 의미하며 런타임에 안전하게 처리되고, 다운캐스팅은 자식 타입에서 부모 타입으로의 자동 변환을 의미한다.
  • 업캐스팅과 다운캐스팅은 모두 상속 관계의 클래스 간 변환으로, 자바 컴파일러가 런타임 오류를 방지하기 위해 언제나 자동으로 캐스팅을 수행해 준다.
  • 업캐스팅은 메모리 크기가 작은 기본형 타입을 큰 타입으로 변환하는 것이고, 다운캐스팅은 큰 크기의 기본형 타입을 작은 타입으로 명시적으로 축소하는 것이다.
정답 보기
①
업캐스팅 = 자식→부모, 자동(항상 안전). 다운캐스팅 = 부모→자식, 반드시 명시적 (ChildType) 캐스트 필요, ClassCastException 가능(Lec6).
④는 기본형의 widening/narrowing(Lec2)으로, 객체 캐스팅과 다른 개념이다.
Q5정의 오버로딩(Overloading)의 정의·규칙으로 옳은 것은?
  • 같은 클래스 내에서 메서드의 이름은 동일하게 유지하지만, 매개변수의 개수나 타입을 다르게 하여 여러 개의 메서드를 정의하는 기법이다.
  • 상속 관계에 있는 부모-자식 클래스 간에, 부모 클래스의 메서드와 완전히 동일한 시그니처를 유지하면서 내부 구현 내용만 재정의하는 기법이다.
  • 같은 클래스 내에서 메서드의 이름과 매개변수 형식은 모두 동일하게 유지하되, 반환 타입(Return Type)만을 다르게 지정하여 여러 개를 만드는 기법이다.
  • 클래스나 메서드 내부에서 사용할 데이터 타입을 미리 지정하지 않고, 외부에서 객체를 생성할 때 타입을 결정하도록 하여 재사용성을 높이는 기법이다.
정답 보기
①
오버로딩은 같은 클래스 내에서 같은 이름·다른 매개변수 형식의 메서드를 정의하는 것이다(컴파일 타임 결정, Lec7).
② 오버라이딩, ③ 반환 타입만 다른 것은 같은 시그니처라 컴파일 에러, ④ 제네릭의 설명이다.
Q6정의 Java 배열(Array)의 정의로 옳은 것은?
  • 서로 다른 데이터 타입의 값들을 하나의 구조로 묶어 가변적인 크기의 메모리 공간에 자유롭게 할당하고 저장할 수 있는 선형 자료구조이다.
  • 동일한 데이터 타입의 값들을 메모리 상에 연속적으로 할당하여 저장하는 자료구조로, 인덱스는 0부터 시작하며 마지막 인덱스는 length - 1이다.
  • 고유한 식별자인 키(Key)와 그에 대응하는 값(Value)을 쌍으로 묶어 저장하며, 키를 통해 값을 매우 빠르게 검색할 수 있도록 설계된 자료구조이다.
  • 기본형 데이터(primitive type)는 저장할 수 없고 객체(Object)만 저장할 수 있으며, 데이터가 추가됨에 따라 동적으로 크기를 늘릴 수 있는 자료구조이다.
정답 보기
②
배열은 동일한 타입의 값을 연속된 메모리에 저장하며, 인덱스는 0부터 시작하고 마지막 인덱스는 length - 1이다. arr.length는 메서드가 아닌 필드다(Lec3).
④는 ArrayList 등 컬렉션(Lec8)의 설명이다.
Q7정의 Unchecked Exception의 정의로 옳은 것은?
  • 코드 작성 시 컴파일러가 try-catch 블록으로 감싸거나 메서드 시그니처에 throws 선언을 하도록 문법적으로 강제하는 예외를 말한다.
  • RuntimeException을 상속받는 계열로, 컴파일 타임이 아닌 런타임에 검사되며 예외 처리(try-catch/throws) 여부가 개발자의 선택 사항인 예외이다.
  • 프로그램의 실행 환경인 JVM 자체에 심각한 문제가 발생했을 때 던져지는 시스템 레벨의 오류로, 애플리케이션 코드에서 절대 잡아서는 안 되는 예외이다.
  • 자바 표준 라이브러리(API)에서 제공하지 않으므로, 개발자가 반드시 Exception 클래스를 직접 상속받아 커스텀 클래스로 구현해야만 하는 예외이다.
정답 보기
②
Unchecked Exception은 RuntimeException 계열로 런타임에 검사되며 처리가 선택 사항이다. 예: NullPointerException, ArrayIndexOutOfBoundsException, ArithmeticException(Lec11).
① Checked Exception, ③ Error의 설명이다.
Q8정의 C++ new/delete의 정의로 옳은 것은?
  • new 연산자는 힙(Heap) 영역에 동적으로 메모리를 할당하고, delete 연산자는 할당된 메모리를 수동으로 해제한다 (배열 할당 시 delete[] 사용).
  • new 연산자는 스택(Stack) 영역에 정적 변수를 생성할 때 사용하고, delete 연산자는 변수가 선언된 스코프가 끝날 때 자동으로 호출되어 메모리를 정리한다.
  • new 연산자로 할당한 객체 메모리는 백그라운드에서 동작하는 가비지 컬렉터(GC)가 주기적으로 검사하여 자동으로 회수하므로 명시적인 delete 호출이 필요하지 않다.
  • new 연산자는 기존에 존재하는 힙 객체의 메모리를 새로운 포인터로 깊은 복사(Deep Copy)할 때 사용하고, delete 연산자는 두 포인터의 메모리 주소를 비교한다.
정답 보기
①
C++ new는 힙 할당, delete는 수동 해제다. 배열은 new int[N]으로 할당하고 반드시 delete[]로 해제해야 한다(단순 delete는 첫 원소만 해제, Lec13).
③은 Java(GC)의 설명으로, C++은 수동 관리다.
Q9정의 C++ 소멸자(Destructor)의 정의로 옳은 것은?
  • 객체가 메모리에 처음 할당되고 생성될 때 컴파일러에 의해 자동으로 호출되어, 내부 멤버 변수들의 초기화 작업을 주로 담당하는 특별한 멤버 함수이다.
  • 객체의 생명 주기가 끝나 메모리에서 소멸될 때 자동으로 호출되어 리소스를 해제하는 멤버 함수로, 클래스 이름 앞에 틸드(~)를 붙여 ~ClassName() 형식으로 선언한다.
  • 자신과 동일한 타입의 다른 기존 객체를 참조(Reference)로 전달받아, 그 객체의 데이터를 기반으로 새로운 객체의 내용을 구성할 때 호출되는 특별한 형태의 생성자이다.
  • 클래스 외부에 정의된 일반 함수(비멤버 함수)임에도 불구하고, 해당 클래스 내부의 private 및 protected 멤버 변수에 자유롭게 접근할 수 있도록 권한을 부여하는 키워드이다.
정답 보기
②
소멸자는 객체의 생명이 끝날 때(스코프 종료·프로그램 종료·delete 시) 자동 호출되어 리소스를 해제하며, 형식은 ~ClassName()다. 소멸 순서는 생성 순서의 역순이다(Lec14).
① 생성자, ③ 복사 생성자, ④ friend의 설명이다.
Q10정의 C++ 위임 생성자(Delegating Constructor)의 정의로 옳은 것은?
  • 초기화 리스트(Initializer List)를 사용하여 동일한 클래스 내에 선언된 다른 생성자에게 멤버 변수의 초기화 작업을 위임하는 형태의 생성자이다.
  • 상속 계층 구조에서 자식 클래스의 객체가 생성될 때, 초기화 리스트를 통해 부모(Super) 클래스의 특정 생성자를 명시적으로 선택하여 호출하는 생성자이다.
  • 클래스를 인스턴스화할 때 전달해야 하는 인수가 전혀 없거나, 모든 매개변수에 기본값이 지정되어 있어서 괄호 없이도 객체를 생성할 수 있게 해주는 생성자이다.
  • 동적으로 할당된 객체의 생명 주기가 끝나거나 delete 연산자가 호출될 때 실행되어, 해당 객체가 점유하던 동적 메모리나 파일 핸들 등을 운영체제에 반환하는 생성자이다.
정답 보기
①
위임 생성자는 초기화 리스트에서 같은 클래스의 다른 생성자를 호출해 초기화를 넘긴다(Foo(int y) : Foo('D', y) { }, Lec14).
③ 기본 생성자, ④ 소멸자의 설명이다.

Section 2 — 정의 → 용어

8문제 · "다음 설명에 해당하는 용어는?"

Q11용어 "Java가 더 이상 참조되지 않는 객체를 자동으로 회수하여 메모리를 관리하는 기능"에 해당하는 것은?
  • Garbage Collector
  • Destructor
  • Iterator
  • Constructor
정답 보기
① Garbage Collector
Java는 자동 메모리 관리로, Garbage Collector가 더 이상 참조되지 않는 객체를 자동 회수한다. C/C++은 new/delete로 수동 관리한다(Lec4).
Q12용어 "break 없이 다음 case로 실행이 계속 이어지는 switch 문의 동작"을 가리키는 용어는?
  • Short-Circuit
  • Fall-through
  • Overflow
  • Casting
정답 보기
② Fall-through
switch에서 break를 빠트리면 다음 case로 실행이 이어지는 것을 fall-through라 한다(Lec3).
① Short-Circuit은 &&/||에서 좌변만으로 결과가 정해지면 우변을 평가하지 않는 것이다.
Q13용어 "내부가 동적 배열로 되어 있어 인덱스 랜덤 접근 get(i)이 O(1)로 빠른" List 구현체는?
  • LinkedList
  • ArrayList
  • HashSet
  • PriorityQueue
정답 보기
② ArrayList
ArrayList는 내부가 동적 배열이라 get(i)이 O(1)로 빠르지만, 중간 삽입/삭제는 O(n)(shift 필요)이다(Lec8).
LinkedList는 노드 체인이라 get은 O(n), 중간 삽입/삭제는 O(1)이다.
Q14용어 "중복을 허용하지 않고(중복 없음), 내부적으로 Red-Black Tree를 써서 요소를 자연 순서로 정렬해 저장하는" 컬렉션은?

  • ArrayList

HashSet은 해시테이블 기반으로 순서 보장이 없고 O(1)이다.

Section 3 — A/B 개념 옳고 그름

6문제 · 두 개념 주장 A와 B 중 옳은 것을 고르시오 — 그럴싸한 오개념 주의

Q19A/B Java Call-by-Value(기본 타입) vs 객체 전달에 대한 주장 중 옳은 것은?
  • 기본 타입은 값이 복사되어 메서드 안에서 변경해도 원본에 영향이 없고, 객체는 주소(참조)가 복사되어 메서드 안에서 내부 상태를 변경하면 원본 객체도 함께 변경된다.
  • 기본 타입은 스택(Stack) 메모리 주소가 직접 복사되므로 메서드 내의 변경이 원본에 반영되며, 객체는 힙(Heap)에 저장된 실제 값이 복사되어 전달되므로 원본 객체에는 영향이 없다.
정답 보기
A가 옳다
기본 타입 = 값 복사 → 메서드 내 변경이 원본에 영향 없음(swap(int,int)은 효과 없음).
객체 = 주소(참조) 복사 → 메서드 내에서 내용을 바꾸면 원본 객체가 변경됨(Lec4).
B는 C++의 참조 전달(Pass-by-Reference)이나 깊은 복사를 교묘하게 섞은 오개념이다.
Q20A/B 증감 연산자 ++i(prefix)와 i++(postfix)에 대한 주장 중 옳은 것은? (int x=1, y=1;일 때)
  • x + ++y는 y를 먼저 2로 증가시킨 뒤 x와 더하므로 3이 반환되고, x + y++는 현재의 y값 1을 먼저 더해 2가 반환된 후 y가 증가한다.
  • x + ++y는 현재의 y값 1을 먼저 x와 더하여 2가 반환된 후 y가 증가하며, x + y++는 y를 2로 먼저 증가시킨 뒤 x와 더하므로 3이 반환된다.
정답 보기
A가 옳다
prefix(++y): y가 먼저 2로 증가 후 덧셈 → x + ++y = 1 + 2 = 3.
postfix(y++): y=1로 덧셈 후 2로 증가 → x + y++ = 1 + 1 = 2(Lec2).
Q21A/B 타입 캐스팅 Widening(확장) vs Narrowing(축소)에 대한 주장 중 옳은 것은?
  • 작은 타입에서 큰 타입으로의 변환(widening)은 데이터 손실이 없어 자동 변환되고, 큰 타입에서 작은 타입으로의 변환(narrowing)은 손실 위험이 있어 반드시 (type) 명시적 캐스트가 필요하다.
  • 작은 타입에서 큰 타입으로의 변환(widening)은 메모리 공간을 새로 확장 할당해야 하므로 명시적 캐스트가 필요하고, 큰 타입에서 작은 타입으로의 변환(narrowing)은 기존 공간 내에 수용되므로 자동 변환된다.
정답 보기
A가 옳다
Widening(확장) = 작은→큰 타입, 자동(byte→short→int→long→float→double).
Narrowing(축소) = 큰→작은 타입, 반드시 명시적 (type) 캐스트 필요(Lec2).
Q22A/B break와 continue에 대한 주장 중 옳은 것은?
  • break는 자신이 포함된 가장 가까운 반복문 전체를 즉시 종료하고 빠져나가며, continue는 현재 진행 중인 반복 회차의 남은 코드를 건너뛰고 다음 반복 조건식으로 이동한다.
  • break는 현재 진행 중인 반복 블록의 나머지 코드만 건너뛰고 다음 반복 회차로 유연하게 이동하며, continue는 특정 예외 조건에 도달했을 때 반복문 전체의 실행을 강제로 즉시 종료시킨다.
정답 보기
A가 옳다
break = 반복문 전체를 즉시 종료.
continue = 현재 반복만 건너뛰고 다음 반복으로 이동(Lec3).
Q23A/B C++ virtual 키워드의 동작에 대한 주장 중 옳은 것은?
  • virtual이 없으면 포인터 타입에 따라 컴파일 타임에 호출할 메서드가 결정(정적 바인딩)되고, virtual이 있어야 런타임에 실제 가리키는 객체의 메서드가 호출(동적 바인딩)된다.
  • virtual이 없으면 런타임에 실제 메모리에 할당된 자식 객체의 메서드가 우선적으로 자동 호출(동적 바인딩)되며, virtual을 붙여야만 포인터 타입인 부모 클래스의 메서드로 고정(정적 바인딩)된다.
정답 보기
A가 옳다
C++은 Java와 달리 virtual이 있어야 동적 바인딩이 된다. virtual 없이 Animal* a = new Cow(); 후 a->eat()은 부모(Animal) 메서드를 호출(정적 바인딩)하고, virtual을 붙이면 실제 객체(Cow) 메서드를 호출한다(★Lab11, Lec14).
B는 Java의 기본 동작과 C++의 키워드를 반대로 설명한 오답이다.
Q24A/B C++ 얕은 복사(Shallow Copy) vs 깊은 복사(Deep Copy)에 대한 주장 중 옳은 것은?
  • 얕은 복사는 포인터 멤버의 주소만 복사해 두 객체가 같은 동적 메모리를 공유하므로 dangling pointer나 double free 문제가 발생할 수 있고, 깊은 복사는 새 메모리를 동적 할당하여 내용을 복사하므로 안전하다.
  • 얕은 복사는 멤버 변수의 값을 하나씩 독립적인 새 메모리에 할당하여 복사하므로 메모리 누수 위험이 없고, 깊은 복사는 포인터의 참조 주소만 재귀적으로 연결하여 메모리를 공유하므로 double free 문제가 발생할 수 있다.
정답 보기
A가 옳다
얕은 복사(묵시적 복사 생성자) = 포인터 멤버의 주소만 복사 → 같은 객체 공유 → dangling pointer·double free 위험.
깊은 복사(명시적 복사 생성자 직접 작성) = 새 객체를 생성하여 복사 → 문제 없음(Lec14).

6문제 (주관식) · "○○를 한 문장으로 정의하시오" 류

Q25서술 Java에서 ==와 .equals()의 차이를 설명하고, String의 불변(immutable) 성질과 String Pool을 한두 문장으로 서술하시오.
정답 보기
==: 메모리 주소(참조)를 비교한다 — 같은 객체를 가리키는지.

.equals(): 객체의 내용(값)을 비교한다 — 문자열 내용이 같은지. 그래서 String 내용 비교는 반드시 .equals()를 써야 한다.

String 불변(immutable): 한 번 생성된 String 객체의 내용은 변경할 수 없다.

String Pool: 리터럴("Hello")로 만든 String은 String Pool에 공유 저장되어 같은 리터럴이면 ==가 true지만, new String("Hello")는 Heap에 새 객체를 만들어 ==가 항상 false다(Lec2/4).
Q26서술 Java try-catch-finally 구문에서 finally 블록의 역할을 한 문장으로 정의하고, catch 블록을 작성하는 순서 규칙(구체적/일반적)을 한 문장으로 서술하시오.
정답 보기
finally: 예외 발생 여부와 관계없이 항상 실행되는 블록으로, 주로 리소스 해제(파일 close 등)에 사용한다.

catch 순서: catch는 위에서 아래로 순서대로 검사되므로 구체적인 예외를 위에, 더 일반적인 예외(Exception 등)를 아래에 써야 한다. 일반 예외를 위에 두면 구체적 예외가 절대 잡히지 않는다(Lec11).
Q27서술 Java에서 커스텀(사용자 정의) 예외를 만드는 방법을, Checked 예외와 Unchecked 예외 각각 어떤 클래스를 상속하는지 포함하여 서술하시오.
정답 보기
커스텀 예외는 예외 클래스를 상속(extends)해 만들고, 보통 생성자에서 super(msg)로 메시지를 부모에 전달한다.

예: class IncorrectFileNameException extends Exception { public IncorrectFileNameException(String msg) { super(msg); } }

Unchecked 예외: RuntimeException을 상속.
예: class CapacityExceedException extends RuntimeException { public CapacityExceedException(String msg) { super(msg); } } (Lec11)
Q28서술 Java 파일 입출력에서 FileReader의 read()가 반환하는 값과 파일 끝(EOF)을 어떻게 판단하는지, 그리고 Scanner로 파일을 한 줄씩 읽는 방법을 서술하시오.
정답 보기
FileReader.read(): 문자 하나를 int로 반환하며, 파일 끝(EOF)에 도달하면 -1을 반환한다. 그래서 while ((c = fr.read()) != -1) { ... } 형태로 읽는다(읽은 int를 (char)로 캐스트해 출력).

Q29서술 Lec12 Clean Code 6원칙의 핵심을 각각 한 줄로 서술하시오.
정답 보기
1. 명확한 이름(Clear Names): 변수·메서드·클래스 이름이 의도를 명확히 드러내야 한다(과도한 축약 금지).

2. 개념당 단어 하나(One Word per Concept): 같은 개념에는 일관된 단어를 쓴다(get/fetch/retrieve 혼용 금지).

3. 함수를 작게(Small Functions): 함수는 한 가지 일만 해야 한다.

4. 부작용 최소화(Minimize Side Effects): 함수가 자신의 범위 밖 변수(공유 상태)를 함부로 바꾸지 않는다.

6. 명확한 주석(Clear Comments): WHAT이 아닌 WHY(왜)를 설명하는 주석만 쓴다(Lec12).
Q30서술 ★Lab C++에서 순수 가상 함수(pure virtual function)가 무엇인지 정의하고, 그것을 가진 클래스가 어떤 클래스가 되는지, virtual·override 키워드와 함께 서술하시오. (Lab11)
