C++ Basics Lecture 13

cout/cin · namespace · 포인터 · 레퍼런스 · 헤더파일

C++란?

C의 확장으로 OOP, 제네릭, 함수형 특성을 추가한 언어.
저수준 메모리 조작 + 고성능이 특징. 임베디드·OS·게임엔진에 사용. 
#include <iostream>

int main() {
    std::cout << "Hello World!" << std::endl;
    return 0;  // main에서는 생략 가능
}
g++ -o main main.cpp   # 컴파일
./main                  # 실행

Namespace

Namespace: 식별자에 범위(scope)를 부여하여 이름 충돌을 방지. Java의 패키지와 유사. 
namespace my_ns {
    int x = 123;
}
std::cout << my_ns::x;  // 123

// using 선언으로 매번 std:: 생략 가능
using namespace std;
cout << "Hello" << endl;

헤더 파일 (.h)

C++에서는 피호출 함수가 호출 함수보다 먼저 나와야 한다 (Java와 다름).
헤더 파일: 함수 시그니처만 선언하여 이 문제를 해결. 
// add.h
#ifndef ADD_H
#define ADD_H
int add(int x, int y);
#endif

// add.cpp
#include "add.h"
int add(int x, int y) { return x + y; }

// main.cpp
#include "add.h"
int main() { cout << add(1, 3); }  // 4
#ifndef / #define / #endif: include guard — 같은 헤더가 여러 번 포함되는 것을 방지.

Macro (#define)

#define PI 3.14              // 상수 매크로
#define ADD(x,y) x+y         // 함수형 매크로 (주의!)

cout << ADD(a,b)/c;  // = a+b/c (NOT (a+b)/c) → 괄호 누락 위험!
함수형 매크로는 연산자 우선순위 문제가 생길 수 있다. 인라인 함수 사용을 권장.

typedef

typedef long miles_t;
typedef std::vector<std::pair<std::string,std::string>> pairlist_t;
pairlist_t pl;  // 긴 타입명을 짧게

포인터 (Pointer)

포인터: 변수의 메모리 주소를 저장하는 변수.
· &: 주소 연산자 (address-of)
· *: 역참조 연산자 (dereference) 
int var = 3;
int* ptr = &var;    // ptr에 var의 주소 저장
cout << ptr;        // 주소 출력 (예: 0x7fff...)
cout << *ptr;       // 3 (역참조: 주소의 값)
*ptr = 10;          // var가 10으로 변경
char *str1, str2; → str1은 char*, str2는 char (포인터 아님!) 
// 힙 메모리 할당/해제
int* ptr = new int;  // 힙 할당
delete ptr;           // 반드시 해제! (Java는 GC가 자동 처리)

레퍼런스 (Reference)

Reference: 기존 변수의 별명(alias). 생성 시 바인딩되며 이후 변경 불가.
Java의 참조와 다름 — C++의 레퍼런스는 포인터처럼 주소를 들고 다니지 않음. 
string food = "Pizza";
string& alias = food;
alias = "Burger";     // food도 "Burger"로 변경

포인터 vs 레퍼런스로 swap

// 포인터 방식
void swap(int* a, int* b) {
    int t = *a; *a = *b; *b = t;
}
swap(&x, &y);   // 호출 시 & 필요

// 레퍼런스 방식 (더 깔끔)
void swap(int& a, int& b) {
    int t = a; a = b; b = t;
}
swap(x, y);     // 호출 시 & 불필요

2D 배열 ★ Lab 10 — 슬라이드에 없음

C++에서 2D 배열을 동적으로 만드는 방법 두 가지.

방법 1: 단일 포인터 (1D 배열로 2D 시뮬레이션)

int* arr = new int[N * N];          // N×N 크기 1D 할당
arr[row * N + col] = value;           // 2D 인덱스 → 1D 변환
delete[] arr;                         // 해제: delete[] (배열은 [])

방법 2: 이중 포인터 (포인터의 포인터)

int** arr = new int*[N];             // N개의 포인터 배열 할당
for (int i = 0; i < N; i++)
    arr[i] = new int[N];             // 각 행 할당

arr[row][col] = value;                // 일반 2D 배열처럼 사용

// 해제: 역순으로
for (int i = 0; i < N; i++)
    delete[] arr[i];                  // 각 행 먼저 해제
delete[] arr;                         // 포인터 배열 해제
배열 해제 시 반드시 delete[] 사용. 단순 delete는 첫 원소만 해제.

포인터와 배열의 관계

배열 변수 자체가 첫 번째 원소의 포인터.
array == &array[0] → 같은 주소 
int array[5] = {9, 7, 5, 3, 1};
int* ptr = array;           // array와 &array[0]는 동일
cout << *ptr;               // 9
cout << *(++ptr);           // 7 — 포인터 산술로 순회
