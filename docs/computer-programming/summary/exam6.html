Section 1 — 용어 → 정의

10문제 · "○○의 정의로 옳은 것은?" — 정의의 정확성으로 구분

Q1정의 캡슐화(Encapsulation)의 정의로 옳은 것은?
  • 자식 클래스가 부모 클래스의 속성과 메서드를 물려받아 확장하며 코드를 재사용하는 것
  • 클래스의 속성(데이터)을 외부로부터 숨기고, 검증된 메서드를 통해서만 접근하도록 하는 것
  • 같은 이름의 호출이 객체의 실제 타입에 따라 서로 다른 방식으로 동작하도록 하는 것
  • 타입 변수(T, E 등)로 클래스나 메서드를 정의해 여러 타입에 안전하게 재사용하도록 하는 것
정답 보기
②
캡슐화는 속성을 숨기고 메서드로만 접근하게 하는 OOP 원칙이다(Lec5).
① 상속(Inheritance), ③ 다형성(Polymorphism), ④ 제네릭(Generics)의 정의다.
Q2정의 다형성(Polymorphism)의 정의로 옳은 것은?
  • "같은 이름"의 호출이 객체에 따라 "다른 방식"으로 동작하는 OOP 특성
  • 객체가 new로 생성될 때 한 번 호출되어 멤버 초기화를 담당하는 특수 메서드
  • 속성을 외부로부터 숨기고 검증된 메서드를 통해서만 접근하게 하는 OOP 원칙
  • 메서드 시그니처만 정의하고 구현은 갖지 않는 다중 구현용 청사진
정답 보기
①
다형성은 poly(많다)+morph(형태) → 같은 이름으로 다른 방식의 동작을 수행하는 특성이다(Lec7).
② 생성자, ③ 캡슐화, ④ 인터페이스의 정의다.
Q3정의 메서드 시그니처(Signature)의 정의로 옳은 것은?
  • 메서드 이름 + 매개변수 타입/순서 + 반환 타입을 모두 합한 것 (Java 기준)
  • 메서드 이름 + 매개변수 타입/순서 (반환 타입은 포함되지 않음)
  • 메서드 이름과 접근 제어자만 합한 것으로, 매개변수는 포함되지 않음
  • 반환 타입 + 접근 제어자 + 예외 목록을 합한 메서드 선언부 전체
정답 보기
②
Java에서 시그니처 = 메서드 이름 + 매개변수 타입/순서이며 반환 타입은 포함되지 않는다(Lec7).
그래서 반환 타입만 다른 메서드는 오버로딩이 아니라 컴파일 에러다.
③은 C 언어 기준(C는 오버로딩 불가)이다.
Q4정의 생성자(Constructor)의 정의·규칙으로 옳은 것은?
  • 클래스 이름과 동일하고, 반환 타입이 없으며, new 시 자동 호출되는 초기화 메서드
  • 클래스 이름과 동일하고 반환 타입이 void이며, 객체 소멸 시 자동 호출되는 메서드
  • 메서드 안에서 현재 객체(인스턴스) 자신을 가리키는 데 쓰이는 참조 변수의 이름
  • 인스턴스가 아닌 클래스 자체에 속해 모든 객체가 공유하며 객체 없이 접근하는 멤버
정답 보기
①
생성자는 클래스 이름과 동일, 반환 타입 없음(void도 아님), new로 객체 생성 시 자동 호출되어 초기화를 담당한다(Lec4).
② void도 아니라는 점이 함정. ③ this, ④ static 멤버의 정의다.
Q5정의 static 멤버의 정의로 옳은 것은?
  • 메서드 내부에서 현재 객체(인스턴스) 자신을 가리키는 데 쓰이며 멤버 구분에 사용되는 참조 변수
  • 인스턴스가 아닌 클래스 자체에 속하는 멤버. 객체 생성 없이 ClassName.member로 접근
  • 숨겨진 private 속성의 값을 외부에서 읽도록 안전하게 반환해 주는 접근자(Getter) 메서드
  • 상속 관계에서 부모가 정의한 메서드를 같은 시그니처로 자식 클래스가 재정의한 것
정답 보기
②
static은 클래스 자체에 속하는 멤버로 객체 생성 없이 ClassName.member로 접근한다. static 메서드는 인스턴스 변수/메서드를 쓸 수 없다(Lec4).
① this, ③ Getter, ④ 오버라이딩의 설명이다.
Q6정의 인터페이스(Interface)의 정의로 옳은 것은?
  • abstract 메서드와 일반(구현된) 메서드를 모두 가질 수 있으나 직접 인스턴스화는 하지 못하는 클래스
  • 메서드 시그니처만 정의하는 청사진. 변수는 묵시적으로 public static final, 한 클래스가 여러 개 구현 가능
  • 속성과 메서드를 함께 정의해 객체를 찍어내는 설계도(blueprint)로, 직접 인스턴스화가 가능한 단위
  • 서로 관련 있는 클래스들을 하나로 묶어 이름 충돌을 막아 주는 그룹(네임스페이스)
정답 보기
②
인터페이스는 메서드 시그니처만 정의하는 청사진으로, 모든 메서드는 기본 abstract, 모든 변수는 public static final이며 한 클래스가 여러 개 구현할 수 있다(다중 상속 대안, Lec6).
① 추상 클래스, ③ 클래스, ④ 패키지의 정의다.
Q7정의 추상 클래스(Abstract Class)의 정의로 옳은 것은?
  • abstract 메서드와 일반 메서드를 모두 가질 수 있고, 직접 인스턴스화 불가한 클래스
  • 모든 메서드가 묵시적 abstract이고 변수가 모두 public static final인 다중 구현용 청사진
  • 더 이상 어떤 자식 클래스도 상속하지 못하도록 완전히 막은 final로 봉인된 클래스
  • 다른 클래스 내부에 선언되어 바깥 클래스의 멤버에 직접 접근할 수 있도록 허용된 중첩 클래스
정답 보기
①
추상 클래스는 abstract 메서드와 일반 메서드를 모두 가질 수 있고 직접 인스턴스화가 불가하며, 서브클래스는 abstract 메서드를 반드시 오버라이딩해야 한다(Lec6).
② 인터페이스, ④ Inner Class의 설명이다.
Q8정의 제네릭(Generics)의 정의·장점으로 옳은 것은?
  • 클래스·메서드를 타입 변수(T, E 등)로 정의해 여러 타입에 재사용하는 기법. 타입 안전성 + 캐스팅 불필요
  • 기존 클래스의 속성과 메서드를 기반으로 새 클래스를 확장해 만드는 코드 재사용 메커니즘으로, 다중 상속은 허용되지 않음
  • 서로 관련 있는 식별자에 범위(네임스페이스)를 부여해 이름 충돌을 방지하고 하나로 묶어 주는 기능
  • 객체의 생명이 끝나 스코프를 벗어날 때 자동 호출되어 점유했던 리소스를 해제해 주는 특수 멤버 함수
정답 보기
①
제네릭은 타입 변수(T, E)로 정의해 다양한 타입에 재사용하는 기법이며, 장점은 컴파일 타임 타입 안전성과 캐스팅 불필요다(Lec7).
② 상속, ③ namespace/패키지, ④ 소멸자의 정의다.
Q9정의 C++의 레퍼런스(Reference)의 정의로 옳은 것은?
  • 다른 변수의 메모리 주소를 값으로 저장하며, 가리키는 대상을 바꿀 수 있는 변수
  • 기존 변수의 별명(alias). 생성 시 바인딩되며 이후 변경 불가
  • 비멤버 함수나 다른 클래스에 private 멤버 접근 권한을 부여하는 키워드
  • 길고 복잡한 타입명에 짧은 별칭을 붙여 주는 타입 별명 선언
정답 보기
②
레퍼런스는 기존 변수의 별명(alias)으로, 생성 시 바인딩되고 이후 바꿀 수 없다(Lec13).
① 포인터, ③ friend, ④ typedef의 정의다.
Q10정의 C++의 복사 생성자(Copy Constructor)의 정의로 옳은 것은?
  • 객체의 생명이 끝나 스코프를 벗어날 때 자동 호출되어 점유한 리소스를 해제하는 특수 멤버 함수
  • 같은 타입의 다른 객체로부터 새 객체를 생성할 때 호출되는 생성자. 형식은 ClassName(const ClassName& other)
  • 인수를 하나도 받지 않거나 모든 매개변수에 기본값이 지정되어 인수 없이 호출되는 기본 생성자
  • 자신의 초기화 작업을 같은 클래스의 다른 생성자에게 떠넘겨 중복을 줄이는 위임 생성자
정답 보기
②
복사 생성자는 같은 타입의 다른 객체로부터 새 객체를 생성할 때 호출되며 형식은 ClassName(const ClassName& other)다(Lec14).
① 소멸자, ③ 기본 생성자, ④ 위임 생성자의 정의다.

Section 2 — 정의 → 용어

8문제 · "다음 설명에 해당하는 용어는?"

Q11용어 "객체를 만들기 위한 설계도(blueprint/template)"에 해당하는 용어는?
  • Object (Instance)
  • Class
  • Method
  • Attribute
정답 보기
② Class
클래스 = 객체를 만들기 위한 설계도(blueprint/template). 이를 기반으로 실제 생성된 것이 Object(Instance)다(Lec4).
Q12용어 "현재 객체(인스턴스)를 가리키는 참조로, 매개변수 이름이 속성과 같을 때 구분에 사용"하는 키워드는?
  • super
  • static
  • this
  • new
정답 보기
③ this
this는 현재 객체를 가리키는 참조로, 매개변수와 속성 이름이 겹칠 때 구분하기 위해 쓴다(this.speed = speed;, Lec4).
① super는 부모 멤버 참조다.
Q13용어 "기존 클래스(부모)를 기반으로 새 클래스(자식)를 만들어, 자식이 부모의 속성과 메서드를 물려받는 메커니즘"은?
  • Inheritance (상속)
  • Encapsulation (캡슐화)
  • Composition (합성)
  • Abstraction (추상화)
정답 보기
① Inheritance
상속은 부모를 기반으로 자식을 만들어 속성과 메서드를 물려받는 메커니즘이다(Is-a 관계, Lec6). Has-a 관계는 합성(멤버 변수)으로 표현한다.
Q14용어 "모든 컬렉션을 동일한 방법으로 순회할 수 있게 하는 인터페이스. hasNext(), next(), remove() 메서드를 가진다"는?
  • Comparator
  • Iterator
  • Collection
  • Map
정답 보기
② Iterator
Iterator는 컬렉션을 동일한 방법으로 순회하게 하는 인터페이스이며 메서드는 hasNext(), next(), remove()다(Lec8).
Q15용어 "key → value 매핑을 저장하며, 키는 유일하고 값은 중복 가능한" 컬렉션 인터페이스는?
  • List
  • Set
  • Map
  • Queue
정답 보기
③ Map
Map은 key → value 매핑을 저장하고 키는 유일, 값은 중복 가능하다(Map<K,V>, Lec8).
List는 순서 있고 중복 허용, Set은 중복 없음, Queue는 FIFO다.
Q16용어 "컴파일러가 처리(try-catch) 또는 선언(throws)을 강제하며, 처리하지 않으면 컴파일 에러가 나는" 예외는?
  • Unchecked Exception
  • Error
  • Checked Exception
  • RuntimeException
정답 보기
③ Checked Exception
Checked Exception은 컴파일 타임에 검사되어 처리 또는 throws 선언이 강제된다(예: IOException, SQLException, Lec11).
Unchecked(RuntimeException 계열)는 처리가 선택 사항이다.
Q17용어 "식별자에 범위(scope)를 부여하여 이름 충돌을 방지하는, Java의 패키지와 유사한 C++ 기능"은?
  • Namespace
  • Header file
  • Macro
  • Pointer
정답 보기
① Namespace
Namespace는 식별자에 범위를 부여해 이름 충돌을 방지하며 Java의 패키지와 유사하다(Lec13).
Q18용어 "비멤버 함수 또는 다른 클래스에게 private/protected 멤버 접근 권한을 부여하는" C++ 키워드는? (우정은 전이되지 않음)
  • virtual
  • static
  • friend
  • override
정답 보기
③ friend
friend는 비멤버 함수나 다른 클래스에 private/protected 멤버 접근을 허용한다. 우정은 전이되지 않는다(A가 B의 friend, B가 C의 friend여도 A는 C의 friend가 아님, Lec14).

Section 3 — A/B 개념 옳고 그름

6문제 · 두 개념 주장 A와 B 중 옳은 것을 고르시오

Q19A/B 오버로딩(overloading)과 오버라이딩(overriding)에 대한 주장 중 옳은 것은?
  • 오버로딩은 같은 클래스 내 매개변수가 다른 메서드들이며 컴파일 타임에 결정된다. 오버라이딩은 상속 관계에서 같은 시그니처로 재정의하며 런타임에 결정된다.
  • 오버로딩은 상속 관계에서 같은 시그니처로 재정의하며 런타임에 결정된다. 오버라이딩은 같은 클래스 내 매개변수가 다른 메서드들이며 컴파일 타임에 결정된다.
정답 보기
A가 옳다
오버로딩 = 같은 클래스·매개변수 다름·컴파일 타임(정적) 결정.
오버라이딩 = 상속·시그니처 동일·런타임(동적) 결정(Lec7 비교표).
B는 둘의 설명이 서로 뒤바뀌어 있다.
Q20A/B 인스턴스 메서드와 static 메서드의 바인딩에 대한 주장 중 옳은 것은?
  • 인스턴스 메서드는 정적 바인딩(컴파일 타임, 변수의 선언 타입 기준), static 메서드는 동적 바인딩(런타임, 실제 객체 타입 기준)이다.
  • 인스턴스 메서드는 동적 바인딩(런타임, 실제 객체 타입 기준)으로 오버라이딩되고, static 메서드는 정적 바인딩(컴파일 타임, 변수의 선언 타입 기준)으로 hiding된다.
정답 보기
B가 옳다
인스턴스 메서드 = Overriding · 런타임 · 실제 객체 타입 기준(동적 바인딩).
static 메서드 = Hiding · 컴파일 타임 · 변수의 선언 타입 기준(정적 바인딩)(Lec6 Overriding vs Hiding 표).
A는 두 바인딩이 뒤바뀌어 있다.
Q21A/B 접근 제어자의 범위에 대한 주장 중 옳은 것은?
  • 좁은 것부터 넓은 순서는 private < default < protected < public이다.
  • 좁은 것부터 넓은 순서는 private < protected < default < public이다.
정답 보기
A가 옳다
접근 범위는 좁음→넓음 순으로 private < default < protected < public이다(Lec5).
default는 같은 패키지까지, protected는 다른 패키지의 서브클래스까지 허용하므로 protected가 default보다 넓다. B는 default와 protected 순서가 뒤바뀌었다.
Q22A/B Comparable과 Comparator에 대한 주장 중 옳은 것은?
  • Comparable은 클래스 내부에 compareTo를 구현해 자연 순서를 정의하고, Comparator는 외부에서 compare를 구현해 커스텀 정렬을 정의한다.
  • Comparable은 외부에서 compare를 구현해 커스텀 정렬을 정의하고, Comparator는 클래스 내부에 compareTo를 구현해 자연 순서를 정의한다.
정답 보기
A가 옳다
Comparable = 클래스 내부에 compareTo 구현 → 자연 순서.
Comparator = 외부에서 compare 구현 → 커스텀 정렬(Lec8).
B는 둘의 역할이 뒤바뀌어 있다.
Q23A/B C++의 객체 생성·소멸 순서에 대한 주장 중 옳은 것은?
  • 멤버/상속 객체의 생성 순서와 소멸 순서는 동일하다.
  • 소멸 순서는 생성 순서의 역순이다(생성 A→B이면 소멸 B→A).
정답 보기
B가 옳다
C++에서 소멸 순서는 생성 순서의 역순이다. 멤버가 A→B 순으로 생성되면 소멸은 B→A 순이다(Lec14 소멸자).

6문제 (주관식) · "○○를 한 문장으로 정의하시오" 류

Q25서술 OOP의 3대 개념(캡슐화·상속·다형성)을 각각 한 문장으로 정의하시오.
정답 보기
캡슐화(Encapsulation): 속성(데이터)을 외부로부터 숨기고 검증된 메서드를 통해서만 접근하게 하는 것.

상속(Inheritance): 자식 클래스가 부모 클래스의 특성(속성·메서드)을 물려받는 것.

다형성(Polymorphism): 같은 이름의 메서드가 서브클래스마다(또는 상황마다) 다르게 동작하는 것.

(Lec4 미리보기 표 · Lec5/6/7 정의)
Q26서술 Java의 와일드카드 세 가지 <?>, <? extends T>, <? super T>의 의미를 각각 한 줄로 정의하시오.
정답 보기
<?>: 임의의 타입(unbounded). 타입과 무관하게 받는다.

<? extends T>: T 또는 T의 서브타입만 허용(상한 와일드카드).

<? super T>: T 또는 T의 슈퍼타입만 허용(하한 와일드카드).

예: <? extends Number>는 Integer·Double 등 Number 하위 타입만, <? super Integer>는 Integer·Number·Object 등 상위 타입만 허용한다(Lec7 와일드카드 표).
Q27서술 ★Lab Bounded Type Parameter(<T extends Foo>)가 무엇이며, 와일드카드 <? extends Foo>와 어떤 점이 다른지 한두 문장으로 서술하시오. (Lab6)
정답 보기
Bounded Type Parameter: 타입 변수 T에 상한을 두어(<T extends Foo>) Foo 또는 그 서브타입만 받게 하고, 그 덕분에 T를 통해 Foo의 멤버에 접근할 수 있게 하는 것.

차이: bounded type parameter는 타입 변수 T 자체를 제한하는 것이고, 와일드카드 <? extends Foo>는 주로 메서드 파라미터용으로 (사실상 읽기 위주로) 쓰인다(Lec7 ★Lab6 표).
Q28서술 throw와 throws의 차이를 각각의 위치와 역할 측면에서 한 문장씩 정의하시오.
정답 보기
throw: 메서드 본문 내에서 예외 객체를 직접 발생시키는 실행문이다(throw new IllegalArgumentException("msg");).

throws: 메서드 선언부에서 이 메서드가 던질 수 있는 (checked) 예외를 선언하여 호출자에게 처리를 위임하는 것이다(void foo() throws IOException).

(Lec11 throw vs throws 표)
Q29서술 C++의 포인터(Pointer)를 정의하고, &(주소 연산자)와 *(역참조 연산자)의 역할을 각각 한 줄로 서술하시오.
정답 보기
포인터: 변수의 메모리 주소를 저장하는 변수다.

& (주소 연산자, address-of): 변수의 주소를 구한다(int* p = &var;).

* (역참조 연산자, dereference): 포인터가 가리키는 주소에 저장된 값을 읽거나 쓴다(*p는 그 값, *p = 10;은 값 변경).

(Lec13 포인터)
Q30서술 SOLID 5원칙 각각의 핵심을 한 줄씩 정의하시오. (S/O/L/I/D)
정답 보기
S — SRP(단일 책임 원칙): 클래스는 변경되어야 할 이유가 하나뿐이어야 한다(하나의 책임).

O — OCP(개방-폐쇄 원칙): 확장에 열려 있고 수정에 닫혀 있어야 한다.

L — LSP(리스코프 치환 원칙): 서브클래스는 언제나 슈퍼클래스를 대체할 수 있어야 한다.

I — ISP(인터페이스 분리 원칙): 사용하지 않는 인터페이스 메서드를 구현하도록 강제해서는 안 된다.

D — DIP(의존 역전 원칙): 구체적인 구현이 아닌 추상(인터페이스)에 의존해야 한다.
