Encapsulation Lecture 05

접근 제어자 · Getter/Setter · 패키지

캡슐화란?

Encapsulation (캡슐화): 클래스의 속성(데이터)을 외부로부터 숨기고, 검증된 메서드를 통해서만 접근하도록 하는 OOP 원칙.

목적:
· Abstraction: 내부 구현 세부사항을 숨기고 공개 인터페이스만 노출
· Defensive Programming: 외부로부터의 무단 변경 방지

접근 제어자 (Access Modifiers)

제어자같은 클래스같은 패키지서브클래스 (다른 패키지)그 외
privateOXXX
(default)OOXX
protectedOOOX
publicOOOO
암기: private < default < protected < public (좁음 → 넓음)
클래스에는 public 또는 default만 사용 가능.
private 클래스는 Inner Class에서만 허용.

Getter / Setter

Getter: private 속성의 값을 반환하는 메서드. 형식: public Type getXxx()
Setter: private 속성에 값을 설정하는 메서드. 형식: public void setXxx(Type xxx)

Java 문법이 아닌 컨벤션(관례). 
private int age;

public int getAge() { return age; }
public void setAge(int age) { this.age = age; }

Getter/Setter 활용 목적

목적방법
읽기 전용getter만 제공, setter 없음
쓰기 전용setter만 제공, getter 없음
입력 검증setter 내에서 유효성 확인 후 설정
접근 추적getter/setter 내에서 로그 기록
// 검증 예시
public void setName(String name) {
    if (name == null || name.equals("")) {
        System.out.println("Name cannot be null or empty");
    } else {
        this.name = name;
    }
}

패키지 (Packages)

Package: 관련 클래스·인터페이스·서브패키지를 묶은 그룹. Java의 네임스페이스 기능.

장점:
· 이름 충돌 방지 (다른 패키지에 같은 이름의 클래스 존재 가능)
· 접근 제어 (default/protected 활용)
· 유지보수성 향상 
// 패키지 선언 (파일 최상단)
package computer;
public class Keyboard { }

// 다른 패키지에서 사용
import computer.Keyboard;       // 특정 클래스 임포트
import computer.*;              // 패키지 전체 임포트

주요 내장 패키지

패키지포함 내용
java.langString, System, Math (자동 임포트)
java.utilArrayList, Scanner, Arrays 등
java.io파일 I/O 클래스들

Inner Class

Inner Class: 다른 클래스 내부에 선언된 클래스.
private 접근 제어자는 오직 Inner Class에서만 사용 가능.

캡슐화의 장점

  • 유연성: 내부 구현을 바꿔도 외부 코드에 영향 없음
  • 재사용성: 캡슐화된 코드는 안전하게 재사용 가능
  • 유지보수성: 한 부분 변경이 다른 부분을 깨지 않음
