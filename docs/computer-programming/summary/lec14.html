클래스 선언 및 멤버 정의

C++에서 멤버 함수는 클래스 내부 또는 외부에서 정의 가능.
외부 정의 시 범위 연산자 :: 사용. 
class Rectangle {
    int width, height;   // private by default
public:
    void set(int, int);
    int area();
};

// 클래스 외부에서 정의 (:: 사용)
void Rectangle::set(int x, int y) { width = x; height = y; }
int  Rectangle::area() { return width * height; }
C++ 클래스의 기본 접근 제어자는 private. struct는 public이 기본.

방식JavaC++

힙 (동적 할당)new Rectangle()Rectangle* p = new Rectangle(3, 4);
멤버 접근. 항상스택: . / 힙 포인터: ->
메모리 해제GC 자동delete p; 수동 필수
Rectangle obj(3, 4);
cout << obj.area();    // 12 — 스택: . 사용

Rectangle* foo = new Rectangle(5, 6);
cout << foo->area();   // 30 — 힙 포인터: -> 사용
delete foo;             // 반드시 해제!

static 멤버

static 멤버 변수는 클래스 외부에서 별도로 정의(초기화)해야 한다.
static 메서드 내에서 this 사용 불가. 
class Counter {
public:
    static int count;
    static void inc() { count++; }
};
int Counter::count = 0;  // 클래스 외부에서 정의 필수!

Counter::inc();   // 클래스명으로 접근

생성자 (Constructor)

멤버 초기화 리스트 (Member Initializer List)

// 방법 1: 생성자 본문에서 초기화
Rectangle(int x, int y) { width = x; height = y; }

// 방법 2: 초기화 리스트 (권장)
Rectangle(int x, int y) : width(x), height(y) { }
초기화 리스트는 생성자 본문보다 먼저 처리됨.
기본 초기화와 초기화 리스트가 충돌하면 초기화 리스트가 우선.

기본 생성자 / 오버로딩 / 위임 생성자

// 기본 생성자: 인수 없음 (또는 모두 기본값)
Rectangle() { width = 5; height = 5; }

// 생성자 오버로딩
Rectangle(int a, int b) { width = a; height = b; }

// 위임 생성자
class Foo {
    Foo(char x, int y) { c = x; n = y; }
    Foo(int y) : Foo('D', y) { }  // 첫 번째 생성자에 위임
};
생성자가 하나도 없으면 컴파일러가 묵시적 기본 생성자를 자동 생성. 다른 생성자를 정의하면 자동 생성 안 됨.

소멸자 (Destructor)

Destructor: 객체의 생명이 끝날 때 자동 호출되어 리소스를 해제.
형식: ~ClassName()
· 스코프 종료, 프로그램 종료, delete 시 호출
· 소멸 순서는 생성 순서의 역순 
class C { A a; B b; };
// 생성: A → B
// 소멸: B → A (역순)

복사 생성자 (Copy Constructor)

Copy Constructor: 같은 타입의 다른 객체로부터 새 객체를 생성할 때 호출.
형식: ClassName(const ClassName& other) 
A a1(7);
A a2(a1);   // 복사 생성자 호출
A a3 = a1; // 초기화 — 복사 생성자 호출 (대입 아님!)

복사 생성자가 호출되는 경우

  • T a = b; 또는 T a(b); — 초기화
  • f(a) — 값으로 전달 (pass-by-value)
  • return a; — 값으로 반환 (return-by-value)

얕은 복사 vs 깊은 복사

구분얕은 복사 (Shallow)깊은 복사 (Deep)
방법묵시적 복사 생성자 (자동 생성)명시적 복사 생성자 직접 작성
포인터 멤버주소만 복사 → 같은 객체 공유새 객체를 생성하여 복사
문제dangling pointer, double free없음
// 깊은 복사 (포인터 멤버가 있을 때)
Student(const Student& s) : person(new Person(*s.person)) { }

복사 생성자 vs 대입 연산자

Test t1, t2;
t2 = t1;       // 대입 연산자 (operator=) 호출 → "Assign"
Test t3 = t1;  // 초기화 → 복사 생성자 호출 → "CopyCon"
T a = b;는 대입이 아닌 초기화이므로 복사 생성자가 호출된다!

friend 키워드

friend: 비멤버 함수 또는 다른 클래스에게 private/protected 멤버 접근 권한 부여.
우정은 전이되지 않음: A가 B의 friend, B가 C의 friend여도 A는 C의 friend가 아님. 
class B {
    int b = 10;
public:
    friend void A::showB(B& x);  // A의 메서드에 접근 허용
};

C++ 상속 & 다형성 ★ Lab 11 — 슬라이드에 없음

C++ 상속 문법

C++ 상속: class 자식 : public 부모 { };
Java의 extends에 해당. 접근 지정자(public)를 반드시 명시. 
class Animal {
public:
    void eat() { cout << "I'm eating generic food.\n"; }
};

class Cow : public Animal {
public:
    void eat() { cout << "I'm eating grass.\n"; }
};

virtual 키워드 — 동적 바인딩의 핵심

virtual 없으면 포인터로 업캐스팅해도 항상 부모 메서드 호출 (정적 바인딩).
virtual 있어야 실제 객체의 메서드 호출 (동적 바인딩). 
// virtual 없을 때
Animal* a = new Cow();
a->eat();  // "I'm eating generic food." ← 부모 호출!

// virtual 있을 때
class Animal {
public:
    virtual void eat() { cout << "I'm eating generic food.\n"; }
};
Animal* a = new Cow();
a->eat();  // "I'm eating grass." ← 동적 바인딩!

override 키워드

Java의 @Override와 동일한 역할.
i) 독자가 오버라이딩임을 알 수 있게 함
ii) 컴파일러가 시그니처 오류를 잡아줌 
class Cow : public Animal {
public:
    void eat() override { cout << "I'm eating grass.\n"; }
};

순수 가상 함수 (Pure Virtual Function)

Pure virtual function: = 0으로 선언. Java의 abstract method에 해당.
순수 가상 함수를 가진 클래스 = 추상 클래스 → 직접 인스턴스화 불가.
서브클래스는 반드시 구현해야 함. 
class Animal {
public:
    virtual void eat() = 0;  // 순수 가상 함수 → Animal은 추상 클래스
};

class Cat : public Animal {
public:
    void eat() override { cout << "I'm eating rat.\n"; }  // 반드시 구현
};
