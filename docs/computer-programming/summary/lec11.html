try-catch-finally

try {
    // 예외 발생 가능한 코드
} catch (FileNotFoundException e) {
    e.printStackTrace();         // 구체적인 예외 먼저!
} catch (Exception e) {          // 더 일반적인 예외는 나중에
    e.getMessage();
    e.toString();
} finally {
    // 예외 여부와 관계없이 항상 실행
    if (file != null) file.close(); // 리소스 해제
}
catch는 위에서 아래로 순서대로 검사. 구체적인 예외를 반드시 위에 써야 한다. 일반 예외를 위에 쓰면 구체적 예외가 절대 잡히지 않는다.

try-with-resources

try-with-resources: AutoCloseable 인터페이스를 구현한 리소스를 자동으로 닫는 구문.
블록이 끝나면 close()가 자동 호출됨. 
// 기존 방식 (finally에서 수동 닫기)
BufferedReader br = new BufferedReader(new FileReader(path));
try { return br.readLine(); } finally { if (br != null) br.close(); }

// try-with-resources (권장)
try (BufferedReader br = new BufferedReader(new FileReader(path))) {
    return br.readLine();
}  // br.close() 자동 호출

// 여러 리소스
try (
    BufferedReader br = new BufferedReader(new FileReader("in.txt"));
    BufferedWriter bw = new BufferedWriter(new FileWriter("out.txt"))
) { bw.write(br.readLine()); }

throw vs throws

구분throwthrows
위치메서드 본문 내메서드 선언부
역할예외 객체를 직접 발생시킴이 메서드가 던질 수 있는 checked 예외를 선언
예throw new IllegalArgumentException("msg");void foo() throws IOException { }
// throw: 예외 직접 발생
if (l1.size() != l2.size()) {
    throw new IllegalArgumentException("Length mismatch");
}

// throws: 호출자에게 처리 위임
static void fun() throws IllegalAccessException {
    throw new IllegalAccessException("demo");
}

커스텀 예외 만들기

// Checked 예외: Exception 상속
public class IncorrectFileNameException extends Exception {
    public IncorrectFileNameException(String msg) { super(msg); }
}

// Unchecked 예외: RuntimeException 상속
class CapacityExceedException extends RuntimeException {
    public CapacityExceedException(String msg) { super(msg); }
}

상속에서의 예외

오버라이딩 메서드는 원본 메서드와 같거나 더 좁은 범위의 예외만 선언 가능.
(더 넓은 예외로 확장 불가) 
class Thrower      { public void print() throws IOException { } }
class MoreThrower extends Thrower {
    @Override
    public void print() throws IOException { }  // OK
    // throws Exception  ← ❌ 더 넓은 예외 불가
}
