OOP 3대 개념

  • 캡슐화: 속성을 숨기고 메서드로만 접근
  • 상속: 자식이 부모의 속성·메서드를 물려받음
  • 다형성: 같은 이름으로 다른 방식 동작

클래스·생성자·this·static

  • Class 설계도 / Object 실제 생성된 것(인스턴스)
  • 생성자: 클래스명과 동일, 반환타입 없음(void 아님), new 시 자동 호출
  • this: 현재 객체 참조 (매개변수=속성 구분)
  • static: 클래스 자체에 속함. static 메서드는 인스턴스 변수 접근 불가
  • 기본값: int=0, double=0.0, boolean=false, 객체=null

접근제어자 (좁→넓)

제어자범위
private같은 클래스
default같은 패키지
protected같은 패키지 + 다른 패키지 자식
public어디서나
⚠️ default < protected (protected가 더 넓음)

오버로딩오버라이딩

매개변수달라야같아야
시점컴파일(정적)런타임(동적)
시그니처 = 이름+매개변수 (반환타입 제외). 반환타입만 다르면 에러

정적 vs 동적 바인딩 (Java)

  • 동적(실제 객체): 인스턴스 메서드 (오버라이딩)
  • 정적(참조 타입): static 메서드(hiding), 필드, private, final
  • ⚠️ Parent p=new Child() → 메서드는 Child, 필드·static은 Parent
  • ⚠️ 상위 메서드 안에서 부른 메서드도 오버라이딩되면 자식 것 호출

추상클래스 vs 인터페이스

  • 추상클래스: abstract+일반 메서드 혼용, 인스턴스화 불가, 단일 상속
  • 인터페이스: 시그니처만, 변수는 public static final, 다중 구현

캐스팅 · instanceof

  • 업캐스팅(자식→부모): 자동, 항상 안전
  • 다운캐스팅(부모→자식): 명시적, ClassCastException 가능
  • ⚠️ 다운캐스팅 = 컴파일 통과 / 런타임에 실패할 수 있음
  • ⚠️ 관계없는 타입끼리는 instanceof·캐스팅 둘 다 컴파일 에러

제네릭 · 와일드카드

  • 제네릭: 타입 변수(T,E)로 재사용. 타입안전+캐스팅 불필요
  • <?> 임의 / <? extends T> T+서브 / <? super T> T+슈퍼
  • ★Lab <T extends Foo> = 타입변수 제한, T로 Foo 멤버 접근
  • 제네릭 메서드 순서: static <T> 반환타입 메서드명

super · final

  • super(): 부모 생성자 호출, 첫 줄. super.m() 부모 메서드
  • 생성자 순서: 부모 → 자식
  • final: class 상속불가 / method 오버라이드불가 / var 재할당불가

예외

  • finally는 항상 실행 · catch는 구체적 타입 먼저(상위 먼저면 에러)

C++ 기본

  • 포인터 주소 저장 / &주소 / *역참조
  • 레퍼런스 별명(alias), 바인딩 후 변경 불가
  • new/delete 수동, 배열은 delete[]
  • #ifndef/#define/#endif 헤더 가드(중복 포함 방지)
  • #define ADD(x,y) x+y → 괄호 함정! · typedef 타입 별칭
  • swap: 포인터 swap(&x,&y) / 레퍼런스 swap(x,y)

C++ OOP

  • 초기화 리스트 : w(x), h(y) (본문보다 먼저)
  • 소멸자 ~Class() 스코프 끝/생성 역순 자동 호출
  • 복사생성자 Class(const Class&) — 초기화·값전달·값반환 시 (레퍼런스/포인터 전달은 X)
  • shallow(주소만→double free) vs deep(새 메모리)
  • static 멤버 외부정의 int C::count=0;
  • struct=기본 public / class=기본 private · this는 포인터(this->)
  • friend: private 접근 허용

C++ 다형성 ★Lab

  • virtual 있어야 동적 바인딩 (Java와 반대, 기본은 정적)
  • virtual void f()=0; 순수 가상 함수(추상)
  • void f() override 재정의 명시
  • 가상 소멸자: base 포인터로 delete 시 자식 소멸자 호출 위해 필수
  • 2D 배열: int** m=new int*[N]; m[i]=new int[N]; 해제 역순 delete[]

자주 틀리는 함정 ⚠️

  • 배열은 .length / List는 .size()
  • throw new MyEx() — new 빠뜨리지 말 것
  • 메서드 반환타입(void) 빠뜨리지 말 것
  • 자원(FileWriter)은 try 밖에서 선언해야 finally서 close
  • 생성자에서 오버라이드 메서드 호출 → 자식 필드 아직 0/null
  • == vs equals · default < protected · 반환타입만 다른 오버로딩=에러
  • narrowing(double→int)은 명시적 캐스트 필요, 소수점 버림

기타 (Lec2~3)

  • 증감: i++ 후위(쓰고 증가) / ++i 전위(증가 후 씀)
  • widening(작→큰) 자동 / narrowing(큰→작) 명시적
  • switch fall-through: break 없으면 다음 case로 흘러감
  • break 반복 종료 / continue 현재 반복만 건너뜀

  • ★Lab Random: new Random(seed) 후 nextInt(n)=0~n-1
