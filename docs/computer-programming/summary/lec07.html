Polymorphism Lecture 07

시그니처 · 오버로딩 · 오버라이딩 · 제네릭 · 와일드카드

다형성이란?

Polymorphism (다형성): "같은 이름"으로 "다른 방식"의 동작을 수행하는 OOP 특성.
poly(많다) + morph(형태) → 여러 형태

시그니처 (Signature)

Signature (메서드 시그니처): 메서드를 식별하는 요소.
Java에서 시그니처 = 메서드 이름 + 매개변수 타입/순서
(반환 타입은 시그니처에 포함되지 않음!) 
int    add(int, int);      // 시그니처: add(int, int)
float  add(float, float); // 다른 시그니처 → OK
double add(int, int);    // ❌ 에러: 같은 시그니처, 반환타입만 다름
C에서는 시그니처 = 이름만. 오버로딩 불가.

다형성 유형 4가지

유형메커니즘결정 시점
Ad-hoc (오버로딩)같은 이름, 다른 매개변수컴파일 타임
Subtype (오버라이딩)같은 시그니처, 다른 클래스런타임
Coercion (캐스팅)타입 변환컴파일/런타임
Parametric (제네릭)타입 변수 사용컴파일 타임

오버로딩 (Overloading)

Overloading: 같은 클래스 내에서 같은 이름이지만 매개변수 형식이 다른 여러 메서드를 정의하는 것. 
class Point {
    void move(int dx, int dy)     { x += dx; y += dy; }
    void move(float dx, float dy) { x += dx; y += dy; }
}

생성자 오버로딩 + this() 호출

class Point {
    Point()                    { this(0, 0); }      // 다른 생성자 위임
    Point(int x, int y)        { this.x = x; this.y = y; }
    Point(float x, float y)    { this((int)x, (int)y); }
}
this() 생성자 호출은 반드시 첫 번째 문장이어야 한다.

모호한 오버로딩 → 컴파일 에러

// test(ColoredPoint, Point)와 test(Point, ColoredPoint) 둘 다 일치
Test.test(cp, cp);  // ❌ Error: ambiguous

오버라이딩 vs 오버로딩 비교

구분OverridingOverloading
위치부모-자식 클래스 간같은 클래스 내
시그니처동일달라야 함
반환 타입동일 (or 공변)달라도 됨
결정 시점런타임컴파일 타임
어노테이션@Override 권장없음
생성자는 상속되지 않으므로 오버라이딩 불가.

제네릭 (Generics)

Generics: 클래스·메서드·생성자를 타입 변수(T, E 등)로 정의하여 다양한 타입에 재사용하는 기법.

장점:
· 타입 안전성: 컴파일 타임에 타입 오류 검출
· 캐스팅 불필요: 꺼낼 때 자동으로 올바른 타입 
// 제네릭 클래스
class Wrapper<T> {
    T obj;
    void add(T obj) { this.obj = obj; }
    T get() { return obj; }
}
Wrapper<Integer> m = new Wrapper<>();
m.add(2);
System.out.println(m.get());  // 2

// 제네릭 메서드
public static <E> void printArray(E[] elements) {
    for (E e : elements) System.out.print(e + " ");
}

와일드카드 (Wildcards)

와일드카드의미사용 예
<?>임의의 타입 (unbounded)타입 무관 읽기 전용
<? extends T>T 또는 T의 서브타입 (상한)Number의 하위 타입만 허용
<? super T>T 또는 T의 슈퍼타입 (하한)Integer의 상위 타입만 허용
// 상한 와일드카드: Integer, Double 등 Number 하위 타입 허용
private static Double sum(ArrayList<? extends Number> nums) { ... }

// 하한 와일드카드: Integer, Number, Object 허용
public static void addNumbers(List<? super Integer> list) { ... }

Bounded Type Parameter ★ Lab 6 — 슬라이드에 없음

Bounded Type Parameter: 타입 변수에 상한을 두어 특정 클래스의 멤버에 접근 가능하게 함.
와일드카드 <? extends T>와 다름 — 이것은 타입 변수 자체를 제한하는 것. 
// T만 쓰면 Worker의 멤버에 접근 불가
public class Scheduler<T> {
    T scheduleFair() {
        T w = workers.get(0);
        w.waitTime;  // ❌ 에러: T가 Worker인지 알 수 없음
    }
}

// T extends Worker: Worker 하위 타입만 허용 + Worker 멤버 접근 가능
public class Scheduler<T extends Worker> {
    T scheduleFair() {
        T w = workers.get(0);
        w.waitTime;  // ✅ Worker 멤버 접근 가능
        return w;
    }
}
구문의미멤버 접근
<T>임의의 타입Object 메서드만
<T extends Foo>Foo 또는 Foo의 서브타입Foo의 멤버 접근 가능
