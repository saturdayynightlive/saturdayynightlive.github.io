Inheritance Lecture 06

extends · overriding · super · interface · abstract class

상속이란?

Inheritance (상속): 기존 클래스(부모/슈퍼클래스)를 기반으로 새 클래스(자식/서브클래스)를 만드는 메커니즘. 자식 클래스는 부모의 속성과 메서드를 물려받는다.

장점: 모듈성, 재사용성, 일관성 
class Child extends Parent {
    // 추가 속성/메서드
}

Is-a vs Has-a 규칙

Is-a (상속): "B is an A"가 성립할 때만 상속 사용.
예) CarOwner is-a Person → extends 사용

Has-a (합성): "B has an A"이면 멤버 변수 사용.
예) CarOwner has-a Car → Car 타입 필드로 가짐

상속 유형

유형설명Java 지원
Single부모 하나O
Multi-levelA → B → C 체인O
Hierarchical부모 하나, 자식 여럿O
Multiple부모 여럿X (diamond problem)
모든 클래스는 암묵적으로 Object를 상속한다.

instanceof 연산자

obj instanceof ClassName: 객체가 해당 클래스(또는 슈퍼클래스)의 인스턴스인지 boolean 반환. 
Child c = new Child();
c instanceof Child   // true
c instanceof Parent  // true (Child is-a Parent)

메서드 오버라이딩 (Overriding)

Overriding: 서브클래스가 슈퍼클래스의 인스턴스 메서드를 동일한 시그니처로 재정의하는 것.
  • @Override 어노테이션 사용 권장 (컴파일 오류 잡아줌)
  • 반환 타입: 동일하거나 공변 서브타입
  • 접근 제어자: 같거나 더 넓어야 함
  • 오버라이딩 불가: private, static, final 메서드

Overriding vs Hiding

구분인스턴스 메서드static 메서드
이름OverridingHiding
결정 시점런타임 (동적 바인딩)컴파일 타임 (정적 바인딩)
기준실제 객체의 타입변수의 선언 타입
Super s = new Sub();
s.greeting(); // static → "Goodnight" (Super 기준)
s.name();     // instance → "Henry"    (Sub 기준, 동적 바인딩)

부모 → 자식Downcasting반드시 명시적 (ChildType)ClassCastException 가능

super 키워드

super: 부모 클래스의 멤버를 참조.
· super(): 부모 생성자 호출 — 반드시 생성자의 첫 번째 줄
· super.method(): 부모 메서드 호출
· super.super.method()는 불가 (조부모 직접 접근 금지)

생성자 체이닝 (Constructor Chaining)

생성자 호출 시 Object까지 자동으로 체인 연결됨. 명시적 super()가 없으면 컴파일러가 자동 삽입.

실행 순서: 조부모 생성자 → 부모 생성자 → 인스턴스 변수 초기화 → 생성자 본문

final 키워드

대상의미

final method오버라이딩 불가 (상속은 가능)

인터페이스 (Interface)

Interface: 메서드 시그니처만 정의하는 청사진(blueprint). 클래스가 아님.
· 모든 메서드: 기본적으로 abstract (본문 없음)
· 모든 변수: 묵시적으로 public static final
· 하나의 클래스가 여러 인터페이스 구현 가능 (다중 상속 대안) 
interface Flyable {
    void fly();  // abstract by default
}
class Bird implements Flyable {
    @Override
    public void fly() { System.out.println("Bird flies"); }
}

추상 클래스 (Abstract Class)

Abstract Class: abstract 메서드와 일반 메서드를 모두 가질 수 있는 클래스.
· 직접 인스턴스화 불가
· 서브클래스는 abstract 메서드를 반드시 오버라이딩해야 함 
abstract class Person {
    String name;
    void printName() { System.out.println(name); }  // 구현됨
    abstract void work();                            // 반드시 오버라이딩
}

Abstract Class vs Interface

구분Abstract ClassInterface
메서드abstract + 구현 모두 가능abstract만 (Java 8+: default/static 허용)
변수모든 종류public static final만
상속/구현하나만 extends여러 개 implements 가능
사용 시기공유 로직 + 공통 인터페이스다중 구현 / 순수 설계 명세

toString() / equals() 오버라이딩

@Override
public String toString() {
    return String.format("%s %s (%d)", fName, lName, id);
}

@Override
public boolean equals(Object o) {
    if (this == o) return true;
    if (o instanceof Student) {
        Student s = (Student) o;
        return this.id == s.id;
    }
    return false;
}
