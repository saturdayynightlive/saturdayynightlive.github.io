TreeSet — Comparator 패턴

TreeSet은 생성 시 Comparator를 주입하여 정렬 기준을 커스터마이징 가능.
여러 기준으로 정렬하려면 Comparator 구현체를 각각 만들어서 사용. 
class NameComp implements Comparator<Employee> {
    public int compare(Employee e1, Employee e2) {
        return e1.getName().compareTo(e2.getName());
    }
}
class IdComp implements Comparator<Employee> {
    public int compare(Employee e1, Employee e2) {
        return e1.getId().compareTo(e2.getId());
    }
}

TreeSet<Employee> byName = new TreeSet<>(new NameComp());
TreeSet<Employee> byId   = new TreeSet<>(new IdComp());

TreeSet 출력 예시

Set set = new TreeSet();
set.addAll(Arrays.asList(2, 1, 3, 6, 2, 4, 7, 0));
System.out.println(set);  // [0, 1, 2, 3, 4, 6, 7] — 정렬, 중복 제거

// HashMap: 삽입 순서 보장 없음

PriorityQueue 동작 방식

PriorityQueue: 내부적으로 Min-Heap 구조. poll() 시 가장 작은 값부터 꺼낸다.
삽입 순서와 무관하게 우선순위 기준으로 정렬된다. 
Queue<Integer> pq = new PriorityQueue<>();
pq.add(3); pq.add(1); pq.add(2);
pq.poll(); // 1 (최솟값)
pq.poll(); // 2
pq.poll(); // 3

Collections 유틸 — 추가 메서드

List list = Arrays.asList("one Two three Four five six one".split(" "));
List sub  = Arrays.asList("Four five six".split(" "));

Collections.indexOfSubList(list, sub);      // 3
Collections.lastIndexOfSubList(list, sub);  // 3
Collections.replaceAll(list, "one", "Yo");
// → [Yo, Two, three, Four, five, six, Yo]
