Collections Lecture 08

List · Set · Map · Queue · Iterator

컬렉션 계층 구조

Collection (interface) ├── List (ordered, duplicates OK) │ ├── ArrayList │ └── LinkedList ├── Set (no duplicates) │ ├── HashSet │ └── TreeSet └── Queue (FIFO) ├── LinkedList ├── Deque (양방향) └── PriorityQueue Map (별도 계층) ├── HashMap └── TreeMap

배열 vs 컬렉션

구분배열 (Array)컬렉션 (Collection)
크기고정가변
타입기본 타입 + 객체객체만
유틸 메서드없음풍부함

List 인터페이스

List: 순서 있는(ordered) 시퀀스. 인덱스로 접근. 중복 허용.
주요 메서드설명
add(e) / add(i, e)끝에 추가 / 인덱스 위치에 삽입
get(i)인덱스 i의 요소 반환
remove(i) / remove(o)인덱스 삭제 / 객체 삭제
size()요소 개수
contains(o)포함 여부
indexOf(o)첫 번째 등장 인덱스
subList(from, to)부분 리스트 반환 (to 미포함)

ArrayList vs LinkedList

연산ArrayListLinkedList
랜덤 접근 get(i)O(1) 빠름O(n) 느림
중간 삽입/삭제O(n) 느림 (shift 필요)O(1) 빠름
내부 구조동적 배열노드 체인 (prev/next 포인터)
LinkedList 추가 메서드: addFirst(), addLast(), getFirst(), getLast(), removeFirst(), removeLast()

Iterator 인터페이스

Iterator: 모든 컬렉션을 동일한 방법으로 순회할 수 있게 하는 인터페이스.
메서드: hasNext(), next(), remove() 
Iterator<Integer> it = list.iterator();
while (it.hasNext()) {
    int i = it.next();
    if (i == 2) it.remove();  // 안전한 삭제
}
for-each 안에서 직접 list.remove()를 호출하면 ConcurrentModificationException 발생!
삭제할 땐 반드시 Iterator를 사용해야 한다.

Set 인터페이스

Set: 수학적 집합. 중복 없음. 순서 보장 없음(HashSet) 또는 정렬(TreeSet). 
// 집합 연산
setA.addAll(setB);     // 합집합 (union)
setA.retainAll(setB); // 교집합 (intersection)
setA.removeAll(setB); // 차집합 (difference)
구분HashSetTreeSet
내부 구조해시테이블Red-Black Tree
순서보장 없음자연 순서 정렬
성능O(1) 삽입/검색O(log n)
조건없음Comparable 또는 Comparator 필요

Comparable vs Comparator

Comparable: 클래스 내부에 compareTo(Object o)를 구현 → 자연 순서 정의
Comparator: 외부에서 compare(T e1, T e2)를 구현 → 커스텀 정렬 
// Comparator로 TreeSet 생성
class NameComp implements Comparator<Player> {
    public int compare(Player e1, Player e2) {
        return e1.getName().compareTo(e2.getName());
    }
}
TreeSet<Player> sortedByName = new TreeSet<>(new NameComp());

Map 인터페이스

Map: key → value 매핑 저장. 키는 유일, 값은 중복 가능. Map<K, V>.
주요 메서드설명
put(k, v)추가/업데이트
get(k)값 조회 (없으면 null)
containsKey(k)키 존재 여부
keySet()모든 키의 Set 반환
구분HashMapTreeMap
순서보장 없음키 기준 정렬
성능O(1)O(log n)

Queue / Deque / PriorityQueue

종류특징주요 메서드
QueueFIFO (선입선출)add(), poll() (제거+반환), peek() (제거없이 확인)
Deque양쪽 끝 삽입/삭제addFirst/Last(), pollFirst/Last(), peekFirst/Last()
PriorityQueue최솟값 먼저 poll (min-heap)자연 순서 또는 Comparator

어떤 컬렉션을 쓸까?

상황선택
키로 검색Map (HashMap)
순서 있는 목록, 빠른 접근ArrayList
순서 있는 목록, 빠른 삽입/삭제LinkedList
중복 제거 + 빠른 검색HashSet
중복 제거 + 정렬TreeSet
FIFO 처리Queue (LinkedList)
우선순위 처리PriorityQueue

Collections 유틸리티

Collections.sort(list);         // 정렬
Collections.reverse(list);      // 역순
Collections.shuffle(list);      // 랜덤 섞기
Collections.min(list);          // 최솟값
Collections.max(list);          // 최댓값
Collections.swap(list, i, j);   // i, j 위치 교환
Collections.fill(list, value);  // 전체를 value로 채우기
